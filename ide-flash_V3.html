1执行模型：SIMT 隐式并行 vs DSA 显式分工最根本的差异
GPU · SIMT隐式
并行单元warp / thread 硬件调度，程序员写单线程视角
计算CUDA core 统一执行，无算/存分离
数据搬运访存延迟由 warp 切换隐藏，自动
流水SM 内隐式重叠，硬件负责
NPU · DSA (达芬奇)显式
并行单元Cube / Vector / MTE 三类单元，需手动派活
计算矩阵走 Cube，逐元素走 Vector，分开编排
数据搬运MTE 显式指令 在 UB↔L1↔HBM 间搬，手写
流水须手动双缓冲，否则单元空转
GPU 的一段 kernel 在硬件层"自动"完成的并行、隐藏延迟、流水重叠，在昇腾上全部要显式写出来。这就是为什么朴素直译只能跑到 ~15%：算子逻辑对了，但没有把搬运和计算编排成流水。

Triton 原语昇腾对应归宿原因

3内存层级：隐式缓存 vs 显式管理迁移工作量的真正来源
GPU 内存层级
Register线程私有，编译器分配隐式
Sharedblock 内共享，手动但灵活半隐式
L2 Cache硬件自动缓存 HBM隐式
HBM全局显存隐式
昇腾内存层级
UBVector 缓冲，容量硬顶 192KB显式

L0A/B/CCube 输入输出缓冲显式
HBM全局，靠 MTE 指令搬显式
GPU 上"分块能不能放下"由缓存自动兜底，写错顶多变慢；昇腾上 UB 是硬容量约束——本算子 BLOCK_M=128 / BLOCK_N=64 / DIM=64，qk 与 out_buffer 均为 fp32（各 32KB），叠加 fp16 的 Q/K/V 分块与 num_stages=4 级多缓冲后峰值 ~226KB，直接溢出 192KB，编译器会拒绝。所以 Tiling 不是优化项而是能否跑通的前提，这是第 4 节第一个鸿沟的根因。
